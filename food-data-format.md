# 食物数据导入格式说明

本文档描述 HealthTracker 应用支持导入的食物数据文件格式。

---

## 支持的文件格式

### 1. JSON 格式（推荐）

```json
{
  "dataSource": "数据来源说明",
  "note": "其他备注信息（可选）",
  "foods": [
    {
      "name": "食物名称",
      "category": "分类",
      "calories": 热量数值,
      "carbohydrates": 碳水化合物数值,
      "protein": 蛋白质数值,
      "fat": 脂肪数值,
      "emoji": "图标"
    }
  ]
}
```

**示例：**

```json
{
  "dataSource": "中国食物成分表",
  "note": "数据来源于中国食物成分表第七版",
  "foods": [
    {
      "name": "大米",
      "category": "主食",
      "calories": 346,
      "carbohydrates": 77.2,
      "protein": 7.4,
      "fat": 0.8,
      "emoji": "🍚"
    },
    {
      "name": "苹果",
      "category": "水果",
      "calories": 52,
      "carbohydrates": 13.5,
      "protein": 0.2,
      "fat": 0.2,
      "emoji": "🍎"
    }
  ]
}
```

### 2. CSV 格式

```csv
name,category,calories,carbohydrates,protein,fat,emoji
大米,主食,346,77.2,7.4,0.8,🍚
苹果,水果,52,13.5,0.2,0.2,🍎
```

---

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | ✅ | 食物名称，如"大米"、"苹果" |
| category | String | ⭕ | 分类，见下方分类建议 |
| calories | Double | ✅ | 热量，单位：kcal/100g |
| carbohydrates | Double | ✅ | 碳水化合物，单位：g/100g |
| protein | Double | ✅ | 蛋白质，单位：g/100g |
| fat | Double | ✅ | 脂肪，单位：g/100g |
| emoji | String | ⭕ | 食物图标（emoji），如"🍚"、"🍎" |

---

## 分类建议

| 分类 | 说明 | 示例 |
|------|------|------|
| 主食 | 米面粮薯类 | 大米、面粉、米饭、面条、馒头、红薯、土豆、玉米、燕麦 |
| 肉类 | 畜禽肉类 | 猪肉、牛肉、羊肉、鸡肉、鸭肉、排骨、肝脏 |
| 海鲜 | 鱼虾贝类 | 草鱼、鲤鱼、虾、螃蟹、扇贝、鱿鱼 |
| 蔬菜 | 各类蔬菜 | 白菜、菠菜、西红柿、黄瓜、茄子、西兰花 |
| 水果 | 各类水果 | 苹果、香蕉、橙子、葡萄、西瓜、草莓 |
| 蛋奶 | 蛋类和乳制品 | 鸡蛋、鸭蛋、牛奶、酸奶、奶酪 |
| 豆类 | 豆类及豆制品 | 黄豆、绿豆、豆腐、豆浆、腐竹 |
| 坚果 | 坚果和种子 | 花生、核桃、杏仁、腰果、瓜子 |
| 油脂 | 食用油类 | 植物油、橄榄油、猪油 |

## 注意事项

1. **数据准确性**：营养数据需为每 100 克可食部的含量
2. **食物名称**：使用中文常见名称，避免使用品牌名或方言
3. **分类一致**：使用本文档定义的分类名称
4. **Emoji 兼容**：使用标准 Unicode emoji
5. **去重**：同一种食物不要重复出现
6. **文件编码**：UTF-8 编码

---

## 导入说明

1. 在应用中进入「设置」→「导入食物数据」
2. 点击「导入 JSON 文件」或「导入 CSV 文件」
3. 选择符合上述格式的文件
4. 系统会自动识别重复食物并跳过

> ⚠️ 注意：重复的食物名称会被自动跳过，已记录的饮食记录不会受影响。
