# 食物营养数据爬虫提示词

本文档用于指导 AI 爬取食物营养数据，生成的数据文件可直接导入 HealthTracker 应用。

---

## 数据格式要求

### JSON 格式（推荐）

```json
{
  "dataSource": "数据来源说明",
  "note": "其他备注信息",
  "foods": [
    {
      "name": "食物名称",
      "category": "分类",
      "calories": 热量数值,
      "carbohydrates": 碳水化合物数值,
      "protein": 蛋白质数值,
      "fat": 脂肪数值,
      "emoji": "图标"
    }
  ]
}
```

### CSV 格式

```csv
name,category,calories,carbohydrates,protein,fat,emoji
大米,主食,346,77.2,7.4,0.8,🍚
猪肉（瘦）,肉类,143,0.2,20.3,6.2,🥩
```

---

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | ✅ | 食物名称，如"大米"、"猪肉（瘦）" |
| category | String | ⭕ | 分类，如"主食"、"肉类"、"蔬菜"、"水果"、"蛋奶"、"豆类"、"坚果"、"海鲜"、"油脂" |
| calories | Double | ✅ | 热量，单位：kcal/100g |
| carbohydrates | Double | ✅ | 碳水化合物，单位：g/100g |
| protein | Double | ✅ | 蛋白质，单位：g/100g |
| fat | Double | ✅ | 脂肪，单位：g/100g |
| emoji | String | ⭕ | 食物图标（emoji），如"🍚"、"🥩"、"🥬" |

---

## 分类建议

| 分类 | 说明 | 示例 |
|------|------|------|
| 主食 | 米面粮薯类 | 大米、面粉、米饭、面条、馒头、红薯、土豆、玉米、燕麦 |
| 肉类 | 畜禽肉类 | 猪肉、牛肉、羊肉、鸡肉、鸭肉、排骨、肝脏 |
| 海鲜 | 鱼虾贝类 | 草鱼、鲤鱼、虾、螃蟹、扇贝、鱿鱼 |
| 蔬菜 | 各类蔬菜 | 白菜、菠菜、西红柿、黄瓜、茄子、西兰花 |
| 水果 | 各类水果 | 苹果、香蕉、橙子、葡萄、西瓜、草莓 |
| 蛋奶 | 蛋类和乳制品 | 鸡蛋、鸭蛋、牛奶、酸奶、奶酪 |
| 豆类 | 豆类及豆制品 | 黄豆、绿豆、豆腐、豆浆、腐竹 |
| 坚果 | 坚果和种子 | 花生、核桃、杏仁、腰果、瓜子 |
| 油脂 | 食用油类 | 植物油、橄榄油、猪油 |

## 数据来源建议

推荐从以下权威数据源爬取：

1. **中国食物成分表**（国家标准）
2. **USDA FoodData Central**（美国农业部）
3. **foodwake.cn**（食物派）
4. **fatsecret.cn**（FatSecret 中国版）

## 爬虫提示词模板

```
请爬取 [网站名称] 的食物营养数据，要求如下：

1. 目标数据：基础食材（菜市场散装食物），不包含包装食品
2. 数据范围：
   - 主食类：大米、面粉、杂粮等
   - 肉类：猪牛羊鸡鸭及各部位
   - 海鲜类：鱼虾蟹贝类
   - 蔬菜类：叶菜、根茎、瓜果类
   - 水果类：常见水果
   - 蛋奶类：蛋类和乳制品
   - 豆类：豆类及豆制品
   - 坚果类：坚果和种子
   - 油脂类：食用油

3. 输出格式：JSON，结构如下：
{
  "dataSource": "[数据来源]",
  "foods": [
    {
      "name": "食物名称",
      "category": "分类",
      "calories": 热量,
      "carbohydrates": 碳水,
      "protein": 蛋白质,
      "fat": 脂肪,
      "emoji": "图标"
    }
  ]
}

4. 数据要求：
   - 营养数据为每100克可食部的含量
   - 热量单位：kcal
   - 碳水/蛋白质/脂肪单位：g
   - 数值保留1位小数

5. Emoji规则：
   - 同种类食物使用相同emoji
   - 参考以下映射：
     - 主食→🍚，肉类→🥩，海鲜→🐟，蔬菜→🥬，水果→🍎，蛋奶→🥚，豆类→🫘，坚果→🌰，油脂→🫒
```

## 注意事项

1. **数据准确性**：营养数据需来自权威来源，确保每100克的数值准确
2. **食物名称**：使用中文常见名称，避免使用品牌名或方言
3. **分类一致**：使用本文档定义的分类名称
4. **Emoji兼容**：使用标准 Unicode emoji，避免使用特殊或新版 emoji
5. **去重**：确保同一种食物不会重复出现
